# Reports

## Root Files
- `latest.md` and `latest.json` are the current status pointers and remain at the reports root.
- `history_manifest.json` and `history_index.md` preserve chronological report discovery.
- `project_progress_recap.md` remains at the reports root for quick project context.

## Phase Folders
- `phase1/` contains Phase 1 validation and foundation reports.
- `phase2/` contains Phase 2 module and workflow reports.
- `phase3/` contains migration, staging, QA, UI, and media reports.
- `phase4/` is reserved for future production/cutover reporting.

Historical reports may also remain at the root for backward compatibility. Do not delete old report files solely because they were copied into a phase folder.
